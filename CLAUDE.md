# CLAUDE.md — 受講生用OSチェックアプリ

## このリポジトリについて

NPO法人 他力本願研究所が運営する「受講生用 思考OSチェック」アプリ。  
GitHub Pages（tariki-labo.github.io/student-check/）でホスティング。  
1ページ完結型SPA（HTML/CSS/JS）。外部フレームワーク不使用。Chart.js（CDN）のみ。

---

## プロジェクト全体の構造

この団体は「朱子学OS」（日本人に深く刷り込まれた儒教由来の思考パターン）のアンインストールを掲げている。  
書籍『もう、親孝行はやめなはれ』の出版を起点に、2つの講座を展開する。

### 関連リポジトリ（すべて tariki-labo.github.io 配下）

| パス | 内容 | 備考 |
|------|------|------|
| `/edo-reiwa/` | 拡散用OSチェック（21問） | 一般向け。書籍の「はじめに」「終わりに」で案内 |
| `/share-tool/` | 読了者シェアツール | 7カテゴリのシェア画像生成。書籍の「終わりに」で案内 |
| `/student-check/` | **受講生用OSチェック（このリポジトリ）** | 講座受講者専用。パスワード付き |

### ファネル上の位置づけ

```
書籍読了 → LINE登録 → プレゼント動画視聴 → 受講生用OSチェック → 個別面談 → 講座申込
```

受講生用OSチェックは、個別面談前にデータを取得するツール。講座終了後にも同じチェックを行い、ビフォーアフターを可視化する。

## 2つのアプリの仕様

### 共通仕様

- **画面遷移：** パスワード → 質問（1問ずつ・シャッフル）→ 名前入力 → 結果（レーダーチャート）
- **回答方式：** 5段階スライダー（0, 25, 50, 75, 100）
  - ラベル：まったくない / あまりない / どちらとも / ややある / よくある
- **スコア算出：** 各設問は2カテゴリの交点を測定。カテゴリスコア ＝ そのカテゴリが含まれる全設問の回答値の平均
- **レーダーチャート：** Chart.js（CDN）、7軸、既存アプリと同じ形式
- **データ送信：** Google Apps Script（GAS）のWeb API経由でスプレッドシートに送信
- **GA4：** 測定ID `G-12W0CMVX96`（既存アプリと同一プロパティ）
- **デザイン：** 和紙テクスチャ風背景、Noto Serif JP（見出し）+ Noto Sans JP（本文）、和風カラーパレット
- **モバイルファースト：** 受講者はスマホからアクセスする想定

### 子育て講座用（os_check_kosodate.html）

- **設問数：** 26問（7つの組み合わせ）
- **パスワード：** `kosodate2026`（ファイル内のCORRECT_PASSWORD定数）
- **7カテゴリ：** 権威服従、課題の混同、自己否定、完璧主義、認知の硬直性、集団同調、世間体・恥の文化
- **ヘッダー：** 「ストップ朱子ハラ連鎖！子育てOS書き換え講座」

### 自分2.0講座用（os_check_jibun20.html）

- **設問数：** 27問（OS測定22問 ＋ 脳の回復段階5問）
- **パスワード：** `jibun20_2026`（ファイル内のCORRECT_PASSWORD定数）
- **出題方式：** OS測定22問と脳の回復段階5問をすべて混ぜてシャッフル（回答者には連続した27問に見える）
- **レーダーチャート：** OS測定22問のみで算出。脳の回復段階5問は含めない
- **脳の回復段階スコア：** 5問の平均値。スプレッドシートにのみ送信（画面には表示しない）
  - スコアが高い＝回復期以前の可能性あり（前川さんが個別面談前に確認する手がかり）
- **ヘッダー：** 「自分らしくラクに生きる！自分2.0講座」

## GAS送信データ形式

```javascript
{
  timestamp: "ISO 8601",
  name: "受講者名",
  course_type: "kosodate" | "jibun20",
  answers: [/* 各設問の回答値 0〜100 */],
  category_scores: [/* 7カテゴリのスコア */],
  brain_recovery_score: 50  // 自分2.0のみ。子育て講座は空文字
}
```

GAS側のコード（前川さんがスプレッドシートに紐づけてデプロイ）：
```javascript
function doPost(e) {
  const sheet = SpreadsheetApp.getActiveSpreadsheet().getActiveSheet();
  const data = JSON.parse(e.postData.contents);
  sheet.appendRow([
    data.timestamp,
    data.name,
    data.course_type,
    ...data.answers,
    ...data.category_scores,
    data.brain_recovery_score || ""
  ]);
  return ContentService.createTextOutput(
    JSON.stringify({ status: "ok" })
  ).setMimeType(ContentService.MimeType.JSON);
}
```

## 設計ルール（絶対遵守）

以下のルールはプロジェクト全体の設計思想に基づく。改修時に必ず遵守すること。

1. **「診断」という言葉は使わない。** 「OSチェック」「OSチェック結果」に統一。医療行為との誤認を避ける。
2. **心理学専門用語は禁止。** 「境界線」「バウンダリー」「パーソナリティ」等は使わない。IT比喩（OS、バグ、アンインストール等）か平易な日本語に翻訳する。
3. **設問文に程度表現を入れない。** 「とても」「どっと」等の副詞は設問に含めない。程度は5段階の回答選択肢で測る。
4. **設問文の語尾は「〜ことがある」等の中立表現。** 「〜してしまう」は測定バイアスになるため原則使わない。
5. **CAT.07は「世間体・恥の文化」で統一。** 他の表記を使わない。
6. **回答データの取り扱い注意表示：** 「回答データは前川にのみ送信されます」を表示する。
7. **フッターには「プライバシーポリシー」と「お問い合わせ」のみ。** 書籍リンク等は置かない。
8. **煽らない。** 高圧的なコンバージョン表現、不安を煽る文言は使わない。

## 改修時の注意

- **設問の追加・変更・削除は慎重に。** 設問データはJavaScriptの `QUESTIONS` 配列内に定義されている。各設問の `cats` プロパティ（カテゴリマッピング）がスコア算出に直結するため、変更時はスコア算出ロジック（`calculateScores` 関数）への影響を必ず確認すること。
- **シャッフルはページロード時に1回実行。** 戻るボタンで前の問に戻れる設計。回答値は `answers` オブジェクトに保持されており、シャッフル順序と独立して管理されている。
- **既存アプリ（edo-reiwa）のデザインとの一貫性を維持。** フォント、カラーパレット、背景テクスチャ、スライダーのスタイルは既存アプリに合わせている。
